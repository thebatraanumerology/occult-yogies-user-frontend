# 🔮 Occult Yogis Webpage

This repository contains the source code for the **Occult Yogis** website.  
It is built using modern frontend technologies to ensure scalability, performance, and maintainability.

---

## 🚀 Tech Stack

This project uses the following technologies:

- **Yarn (Latest Version)** – Package manager for dependency management
- **TypeScript** – Strongly typed JavaScript for better developer experience
- **Tailwind CSS** – Utility-first CSS framework for fast UI development

---

## 📁 Project Overview

This repository is specifically designed for building and maintaining the **Occult Yogis webpage**.

It includes:
- Structured frontend architecture
- Type-safe development using TypeScript
- Pre-configured Tailwind CSS for styling
- Optimized development and production workflows

---

## 🚀 Branch Introduction

This project uses the following branches:

- **main Branch** – Contains the main codebase and production-ready features
- **test Branch** – Contains experimental features and testing code
- **dev Branch** – Contains development of features 


## 📂 Directory Structure

Here is an overview of the project's folder structure to help you navigate:

```text
occult-yogis/
├── public/               # Static assets that don't need processing
├── src/                  # Application source code
│   ├── assets/           # Images, fonts, and other local assets
│   ├── components/       # Reusable UI components
│   ├── context/          # React context providers for global state
│   ├── layout/           # Page layouts and wrappers
│   ├── pages/            # Page-level components (routes)
│   ├── services/         # API calls and external integrations
│   ├── App.tsx           # Main application component
│   └── main.tsx          # Application entry point
├── index.html            # Main HTML template
├── package.json          # Node.js dependencies and scripts
├── tailwind.config.ts    # Tailwind CSS configuration
├── tsconfig.json         # TypeScript configuration
└── vite.config.ts        # Vite configuration
```
